An assembly line that machines an idea into verified code. One core belief: reliability comes from the system, not the model — sequence is driven by commands, scope is hard-blocked by hooks, loops have a stop-loss, and acceptance is judged by an independent validator. The Fable 5 upgrade teaches this line to remember and to converge on its own — without removing a single guardrail.

01

Design Philosophy

Why it works

Four pillars. Each one answers the same question: when AI writes far more code than humans can review, what actually stops the errors?

PILLAR A

Deterministic control

Sequence = the /feat-* commands you invoke one by one. Write scope = scope-track.sh hard-blocking by .active. Loop = a retries counter that stops at loopMaxRetries. Anything that plain deterministic code can guarantee is never delegated to the model's judgment (CLAUDE.md Rule 9).

PILLAR B

File-based handoff

Every subagent starts in a fresh, isolated context; the only thing shared is the filesystem. story.md → brief.md → backend-summary.md → verification.md — handoffs are cheap, reviewable, and resumable, and they keep each unit of work small enough to run on the cheapest capable model.

PILLAR C

Independent verification

"A self-graded paper is worthless." The validator has no Edit/Bash tools — it can only read and report. The test-verifier writes acceptance tests independently from the story's criteria. A model grading its own output favors its own prior reasoning — so the grader is always a different context.

PILLAR D

Fail-Loud honesty

Every agent ends every report with three sections: ✅ Verified / ⚠️ Skipped-Uncertain / ❓ Needs-human-input. "Tests pass" is never allowed to mask skipped tests. Better to be noisy about uncertainty than quiet about delivery.

02

The Assembly Line

The pipeline

A feature from intake to shipping. ⏸ yellow gates are human approvals — the line fully stops there. FABLE5 violet marks the stations added by this upgrade; everything else is the original line. Each station writes its output as a file under <artifactsDir>/<slug>/.

/feat-init→ project.json (once per repo)

Detects the stack and tracks (backend/frontend) and writes the manifest. On a greenfield repo with no build files, it simply asks you. Writes configuration only — never scaffolds code.

/feat-new <slug> "description"→ idea.md + state.json

Opens the feature's working folder and records the raw request.

/feat-researchFABLE5 enhanced→ research.md

First reads MEMORY.md and carries past lessons in as "Prior knowledge" (leads to verify, not facts to trust), then maps the codebase with the read-only Explore agent: relevant files, existing patterns, risks.

/feat-story⏸ human approval→ story.md

The story-writer produces a user story with testable acceptance criteria. You must personally review and approve before the line continues — the first judgment gate.

/feat-spec⏸ human approval→ brief.md

The spec-writer turns the approved story into a technical brief: data model, API contract, files to change, risks. The second judgment gate.

/feat-backend→ code + backend-summary.md

The backend-builder implements per the brief with unit tests, and writes the API contract summary the frontend will consume. Hooks lock it inside the backend directories. (Skipped automatically if no backend track.)

/feat-frontend→ UI + component tests

The frontend-builder implements against the contract. Anything the contract is missing gets reported as a gap — never an invented endpoint.

/feat-shipFABLE5 newoptionalpairs with /goal

Hands the mechanical 06→08 loop to /goal for autonomous convergence: the command prints an exact goal condition for you to paste; an independent evaluator (default: Haiku) judges each turn. Prefer manual? Just run 06→08 yourself as before.

/feat-verify→ verification.md

The test-verifier writes an acceptance test for every story criterion and runs them, recording pass/fail per criterion. A test that cannot fail is not allowed to exist.

/feat-validate→ validation.md

The validator reviews read-only: acceptance, security, scope, conventions. Findings are graded 🔴 Critical / 🟠 Important / ⚪ Minor, each with path:line. Clean → .active is removed; next stop is /feat-docs for user-facing documentation.

/feat-fixbounded loopretries ≤ loopMaxRetries(3)

Routes 🔴🟠 findings and red tests back to the owning track's builder for surgical fixes, then returns to 06 for re-verification. At the cap it marks blocked and stops for human judgment — the loop can never run away.

↺ after a fix, flow returns to /feat-verify → /feat-validate; at the 3-retry cap → blocked, handed to a human
/feat-docs→ README + docs/ (EN → zh-TW)

The doc-writer authors user-facing documentation from what was actually built: updates the README, writes usage/example guides under docsDir (with Mermaid diagrams), English first then _zh-TW translations, language-switch links atop both. Hooks confine it to docsDir and the root README files.

/feat-distillFABLE5 new→ MEMORY.md updated

The memory-distiller banks the lessons that actually happened in this feature into the cross-feature MEMORY.md (Verified facts / General rules / Watchlist). Failures get distilled too: a blocked feature's open findings go to the Watchlist so the next feature doesn't rediscover them. Zero new lessons is a legitimate result — padding is forbidden.

At any time, /feat-status <slug> shows where a feature sits on the line and what to run next.

Under the Hood

Mechanisms
MechanismWhat it doesKey facts
Subagents.claude/agents/*.md (8 files) One agent file per role: frontmatter declares tools and model, the body is its system prompt. Each starts in a fresh isolated context and hands off only through files. Tools are permissions — the validator has no Edit/Bash, so it cannot modify code even if it wanted to.

Hooks.claude/hooks/*.sh (3 files) scope-track.sh hard-blocks out-of-scope writes per .active; protect-secrets.sh blocks writing/committing .env and key files; quality-gate.sh runs typecheck→lint→test per the manifest. Requires jq; without it hooks fail safe (allow) with a notice — i.e. enforcement is OFF. With no .active file, normal editing is never blocked.
Manifest.claude/factory/project.json One per repo: tracks, dirs, commands, protectedDirs, loopMaxRetries. Every command, agent and hook reads it — the same template serves full-stack, frontend-only and backend-only repos. Generated by /feat-init; scripts you don't have yet (e.g. lint) stay blank and the quality gate skips them.
/goal convergence FABLE5Claude Code built-in · paired by /feat-ship The user sets a goal condition; after each turn an independent evaluator (default: Haiku) judges pass/fail and, on fail, feeds the reason into the next turn. The evaluator sees only the conversation, never the filesystem — so every cycle must paste its evidence. /goal is part of the hooks system: disableAllHooks kills it AND scope enforcement — never run the factory with hooks disabled.

Every entry must trace to something that actually happened; hard cap ~150 lines. Lives in the repo, under version control — review memory diffs like code: bad memory compounds exactly as fast as good memory.
04

What Fable 5 Changed

The upgrade

Four things, all tagged FABLE5 throughout the files — one git diff shows every change. The principle: evolve, don't rebuild — the two ⏸ gates, hook enforcement and the bounded loop are untouched.

① Model routing

Orchestrator and workers are billed separately:

RoleModel
Orchestrating session (yours)Fable 5
All 8 agentssonnet (pinned in frontmatter)
/goal evaluatorHaiku (built-in default)
Security-sensitive workswitch that agent to model: opus

② Classifier refusal ≠ defect

Fable 5 ships safety classifiers that can decline a request (API returns stop_reason: "refusal"). A loop that can't tell "the classifier said no" from "my code is wrong" burns its retry budget on rephrasing. Therefore:

  • Agents record refusals as classifier-refusal — working around them is forbidden
  • The validator files them under a separate ⛔ heading (owner: human)
  • /feat-fix step 0 triages: all-refusals → blocked-classifier, no retries consumed

③ Memory layer

Without it, every feature's lessons evaporate with its artifacts. /feat-distill banks lessons after validation passes (or when blocked); /feat-research consults them at the start. The line goes from "reset every time" to "resume" — this is where the compounding actually lands.

④ /goal convergence loop

/feat-ship lets verify→validate→fix converge autonomously; the goal condition carries three hard constraints:

  • Every pass condition demands output actually shown in the conversation as evidence
  • The condition carries its own turn cap (soft); loopMaxRetries remains the hard stop
  • blocked / blocked-classifier / ⛔ are defined as "goal cannot be met" — loud termination, never spinning
Inert-but-harmless by design: these upgrades work — or idle safely — under any other model's session (Opus, Sonnet). The model field is plain routing (saves money), memory and /goal are model-agnostic, and the classifier path simply never triggers off Fable. So the whole team maintains exactly one template — pick the model with /model, not by swapping templates.

References: this upgrade was inspired by @0xCodez's self-improving-agent thread (BlockTempo zh-TW translation). The thread is inspiration, not specification — every product-behavior claim was verified against Anthropic's official documentation before adoption.

06

House Rules

Non-negotiables
The line = Default Mode

Only features you've decided to ship enter the factory. Prototypes/spikes/POCs run in EXPLORE Mode and never go through /feat-* — so test discipline never fights exploration.

Gates are for humans to hold

Actually read story and spec at the ⏸ gates. The day you notice yourself approving without reading — that's the signal to redesign the cadence, not a time-saving trick.

MEMORY.md is code

It's version-controlled and its diffs get reviewed. Bad memory is read by every subsequent feature — compounding cuts both ways.

No workarounds, no varnish

Hooks blocking you is intentional; "tests pass" never masks skipped tests; goals are met by evidence, not narration. Being loud about uncertainty is this line's only religion.
